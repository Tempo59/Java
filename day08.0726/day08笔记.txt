二、回顾：
  1.封装：
	成员私有化，提供共有方法访问私有成员

  2.private关键字：访问权限修饰符

  3.访问类中私有成员的方法：
	获取属性的值：
	getXx方法：需要有返回值，不需要参数

	修改属性的值：
	setXx方法：不需要返回值，需要参数

  4.this关键字：
	this.属性：表示当前对象的属性
	this.方法：表示调用当前对象的方法
	this()：表示调用本类中的构造方法

  5.static关键字：
	static修饰属性：静态属性，不依赖于对象，可以不创建对象直接通过类名访问
	static修饰方法：静态方法，不依赖于对象，可以不创建对象直接通过类名调用
	static修饰代码块：静态代码块，类加载时自动执行一次
		动态代码块：构造代码块，创建对象时执行

		局部代码块：在方法体中的代码块，相当于没有代码块

	public void fun(){
		{int a = 3;}//int a = 3;
	}

  6.单例模式：
	需求：某一个类只需要一个实例

	实现：
	私有化构造方法
	共有静态方法获取本类的对象
	本类中需要有静态的本类对象作为属性

	饿汉式：声明本类对象的同时初始化
	懒汉式：先声明，在共有静态方法中初始化对象

	思考：是否可以在类外将单例对象设置为null
		答：不可以，因为在类外只能是一个引用指向单例对象的堆内存，在单例类中的引用不会收到任何影响


三、面向对象特征--继承：
  什么是继承
  1.什么是继承：
	和类有关，在原有类的基础上，派生出一个新的类，在新的类中可以访问原有类中所有的非私有成员，同时可以添加新的成员的过程叫做类的继承

  2.如何实现继承：
	extends关键字：使用此关键字实现原有类和新类之间的继承关系
	原有类：父类，基类，根类，超类
	新的类：子类
	关系：子类继承父类

	继承的语法：
	//父类
	public class ParentClass{
		//成员
	}

	//子类
	public class ChildClass extends ParentClass{
		//可以访问父类中所有非私有的成员
	}

  3.继承过程中注意事项：
	1）子类只能继承父类中非私有的成员
	2）一个子类只能有一个父类（类单继承）
	3）一个父类可以有多个子类
	4）父类的引用可以指向子类的实例（方法的参数或返回值）
		父类的引用不可以访问子类中独有的成员

  4.不可以被子类继承的父类成员：
	1）父类中私有的成员；
	2）父类的构造方法，但是在子类的构造方法中默认第一条语句是调用父类的无参构造方法

	案例：设计一个动物类（Animal），有姓名，年龄属性，eat方法
		设计一个狗类，继承动物类，设计一个咬人的方法（bit）
		设计一个猫类，继承动物类，设计一个抓人的方法（zhua）

五、方法重写：
	案例：王思聪他爸给5亿，赚
		我爸给我5块，赚

  1.什么是方法重写：
	当子类从父类继承来的方法无法满足自己的需求时，此时我们可以在子类中对方法进行重新定义，将继承来的方法进行完善

  2.方法的重写的规则：
	Override
	首先实在继承过程中
	1）子类中的方法访问权限大于等于父类中的方法
	2）子类中重写的方法的返回值类型小于等于父类方法的返回值
	3）子类中重写的方法的参数小于等于父类方法参数
	4）子类中的方法名必须和父类中的方法名完全一致

	方法的重载：Overload
	在一个类中，方法名相同，参数的个数或参数的类型不同

	注意：当子类重写了父类中的方法时，只要有关子类的对象调用方法时，就会执行重写后的方法

	思考：是否所有的类都可以被继承？
		是否所有的方法都可以被重写？


六、final关键字：
	1）final修饰变量：是常量
		修饰全局变量：是常量属性

	2）final修饰方法：可以被继承，但不能被重写

	3）final修饰类：不可以被继承，没有子类

super关键字：
	super.属性：访问父类中的属性，当子类中声明了与父类中的同名属性时，在子类中需要使用super.属性访问父类中的属性
	super.方法：调用父类的方法，重写过程中，若有调用父类中为重写方法的需求时，需要使用super.方法
	super()：表示调用父类的构造方法，super()是调用父类的无参构造方法，必须是子类构造方法中第一条语句
